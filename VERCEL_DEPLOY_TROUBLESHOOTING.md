# Решение проблемы: не вижу кнопку Deploy в Vercel

## Возможные причины и решения

### ✅ Вариант 1: Vercel автоматически начал деплой

**Что происходит:**
После импорта репозитория Vercel может автоматически начать деплой без кнопки.

**Как проверить:**
1. Посмотрите в верхней части страницы
2. Ищите статус деплоя: "Building...", "Deploying..." или "Ready"
3. Или перейдите на вкладку **"Deployments"** в левом меню

**Что делать:**
- Если видите процесс деплоя - просто дождитесь завершения ✅
- Если деплой уже завершен - найдите URL приложения вверху страницы

--- 

### ✅ Вариант 2: Кнопка находится в другом месте

**Где искать:**

1. **Вверху страницы проекта:**
   - После импорта вы должны быть на странице проекта
   - Ищите кнопку **"Deploy"** или **"Redeploy"** в правом верхнем углу

2. **В настройках проекта:**
   - Перейдите в **Settings** → **General**
   - Прокрутите вниз до секции **"Deployments"**
   - Там может быть кнопка **"Redeploy"**

3. **На вкладке Deployments:**
   - В левом меню нажмите **"Deployments"**
   - Вверху страницы должна быть кнопка **"Redeploy"** или **"Deploy"**

---

### ✅ Вариант 3: Нужно сначала сохранить настройки

**Что делать:**

1. Убедитесь, что вы добавили переменные окружения
2. После добавления переменных может появиться кнопка **"Save"** или **"Continue"**
3. Нажмите её, затем должна появиться кнопка **"Deploy"**

---

### ✅ Вариант 4: Использовать Dashboard

**Альтернативный способ:**

1. Перейдите на главную страницу Dashboard Vercel
2. Найдите ваш проект `knowflow` в списке
3. Нажмите на название проекта
4. Вы попадете на страницу проекта
5. Вверху страницы найдите кнопку **"Deployments"**
6. Нажмите на неё
7. Вверху страницы Deployments должна быть кнопка **"Redeploy"** или **"Deploy"**

---

### ✅ Вариант 5: Деплой через Git push

**Самый простой способ:**

Vercel автоматически деплоит при каждом push в GitHub:

1. Убедитесь, что переменные окружения добавлены:
   - Settings → Environment Variables
   - Добавьте все 4 переменные

2. Сделайте любой commit и push:
   ```bash
   git add .
   git commit -m "Trigger deployment"
   git push origin main
   ```

3. Vercel автоматически начнет деплой
4. Проверьте вкладку **"Deployments"** - там будет виден процесс

---

## Пошаговая инструкция (если ничего не помогло)

### Шаг 1: Проверьте текущее состояние

1. Откройте https://vercel.com/dashboard
2. Найдите проект `knowflow` в списке
3. Нажмите на него

### Шаг 2: Проверьте переменные окружения

1. В левом меню нажмите **"Settings"**
2. Выберите **"Environment Variables"**
3. Убедитесь, что все 4 переменные добавлены:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CRON_SECRET`

### Шаг 3: Проверьте Deployments

1. В левом меню нажмите **"Deployments"**
2. Посмотрите, есть ли там деплои:
   - Если есть деплой со статусом "Ready" - всё готово! ✅
   - Если есть деплой "Building..." - дождитесь завершения
   - Если нет деплоев - перейдите к Шагу 4

### Шаг 4: Запустите деплой через Git

```bash
# В терминале проекта
git add .
git commit -m "Initial deployment"
git push origin main
```

После push:
1. Вернитесь в Vercel Dashboard
2. Откройте вкладку **"Deployments"**
3. Вы увидите новый деплой в процессе

## Быстрое решение

**Самый надежный способ:**

1. Убедитесь, что переменные окружения добавлены (Settings → Environment Variables)
2. Сделайте commit и push:
   ```bash
   git commit --allow-empty -m "Trigger Vercel deployment"
   git push origin main
   ```
3. Vercel автоматически задеплоит
4. Проверьте вкладку **"Deployments"**

---

## Если ничего не помогает

1. **Попробуйте другой браузер** или режим инкогнито
2. **Очистите кеш браузера** (Ctrl+Shift+Delete)
3. **Обновите страницу** (F5 или Cmd+R)
4. **Проверьте, что вы вошли в правильный аккаунт Vercel**
